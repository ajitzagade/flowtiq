# Flowtiq Mobile

React Native mobile app that wraps the Flowtiq web portal in a WebView with native device capabilities: push notifications, secure token storage, file picking, camera, deep linking, and offline detection.

## Prerequisites

- Node.js >= 18
- pnpm >= 8
- Xcode 15+ (for iOS)
- Android Studio + Android SDK (for Android)
- CocoaPods (for iOS pods)
- Ruby + Bundler (for iOS Fastlane)

## Setup

### 1. Initialize the native project

If starting from scratch, run the React Native CLI to generate the native project files:

```bash
cd /path/to/flowtiq-monorepo
npx @react-native-community/cli init FlowtiqMobile \
  --template react-native-template-typescript \
  --directory apps/mobile \
  --skip-install
```

Then copy the custom source files from this repo into the initialized project.

### 2. Apply tenant config (required before building)

The tenant config system sets app name, bundle ID, and web portal URL across all native files.

```bash
# From repo root — apply Vastudeep config (for production builds)
pnpm config:tenant vastudeep

# Apply the development default config
pnpm config:tenant flowtiq
```

This script reads `configs/build/tenant-configs/<slug>.json` and updates:
- `apps/mobile/.env` — env vars for react-native-config
- `apps/mobile/android/app/build.gradle` — applicationId
- `apps/mobile/android/app/src/main/res/values/strings.xml` — app_name
- `apps/mobile/ios/FlowtiqMobile/Info.plist` — CFBundleDisplayName, CFBundleIdentifier

**To add a new tenant:** Create `configs/build/tenant-configs/<slug>.json` (copy `vastudeep.json` as template), fill in all values, then run `pnpm config:tenant <slug>`.

### 3. Environment variables

The `.env` file is generated by `pnpm config:tenant` and is gitignored. An `.env.example` documents the expected shape:

| Variable | Description | Example |
|---|---|---|
| `TENANT_WEBVIEW_URL` | Full URL to the tenant web portal | `https://flowtiq-admin.vercel.app` |
| `TENANT_SLUG` | Tenant identifier | `vastudeep` |
| `TENANT_NAME` | Tenant display name | `Vastudeep Flowtiq` |
| `TENANT_DOMAIN` | Domain for deep link verification | `flowtiq-admin.vercel.app` |

### 4. Firebase configuration

Push notifications require Firebase credentials. These are **per-tenant** and never committed.

**Android:** Place `google-services.json` at:
```
apps/mobile/android/app/google-services.json
```
Download from: Firebase Console > Project Settings > Your App (Android)

**iOS:** Place `GoogleService-Info.plist` at:
```
apps/mobile/ios/GoogleService-Info.plist
```
Download from: Firebase Console > Project Settings > Your App (iOS)

Placeholder files are committed to allow the project to compile without real credentials. FCM will not work until these are replaced.

### 5. Install pods (iOS only)

```bash
cd apps/mobile/ios && pod install && cd ..
```

### 6. Run the app

```bash
# iOS
pnpm --filter @flowtiq/mobile ios

# Android
pnpm --filter @flowtiq/mobile android
```

### 7. Start Metro bundler

```bash
cd apps/mobile && npx react-native start
```

## Android Release Build (Story 4.2)

### Generating the release keystore

Run once to create the Flowtiq signing keystore. Store it securely — never commit it.

```bash
keytool -genkey -v -keystore release.keystore \
  -alias flowtiq \
  -keyalg RSA \
  -keysize 2048 \
  -validity 10000 \
  -storepass YOUR_STORE_PASSWORD \
  -keypass YOUR_KEY_PASSWORD \
  -dname "CN=Flowtiq, OU=Mobile, O=Flowtiq, L=Mumbai, ST=Maharashtra, C=IN"
```

### Encoding the keystore for GitHub Actions

```bash
base64 -i release.keystore | pbcopy  # macOS — copies to clipboard
```

Paste the result as the `ANDROID_KEYSTORE_BASE64` GitHub secret.

### Extracting the SHA-256 fingerprint (for assetlinks.json)

```bash
keytool -list -v -keystore release.keystore -alias flowtiq -storepass YOUR_STORE_PASSWORD
```

Copy the `SHA256:` line from the output. Remove the colons and use as `PLACEHOLDER_SHA256_FINGERPRINT` in `apps/admin-portal/public/.well-known/assetlinks.json`.

### Building a signed AAB locally

```bash
# First apply tenant config
pnpm config:tenant vastudeep

# Then build
cd apps/mobile/android
KEYSTORE_FILE=app/release.keystore \
KEYSTORE_PASSWORD=YOUR_STORE_PASSWORD \
KEY_ALIAS=flowtiq \
KEY_PASSWORD=YOUR_KEY_PASSWORD \
./gradlew bundleRelease
```

Output: `apps/mobile/android/app/build/outputs/bundle/release/app-release.aab`

Or via pnpm script (set env vars first):

```bash
pnpm --filter @flowtiq/mobile android:release
```

### GitHub Actions secrets (Android)

| Secret name | Description |
|---|---|
| `ANDROID_KEYSTORE_BASE64` | Base64-encoded keystore file content |
| `ANDROID_KEYSTORE_PASSWORD` | Keystore store password |
| `ANDROID_KEY_ALIAS` | Key alias within the keystore |
| `ANDROID_KEY_PASSWORD` | Key password |
| `GOOGLE_SERVICES_JSON` | Raw JSON content of google-services.json |
| `GOOGLE_PLAY_SERVICE_ACCOUNT_JSON` | Google Play service account JSON for automated AAB upload |

### Google Play submission

The CI pipeline uses the `r0adkll/upload-google-play` GitHub Action to upload the signed AAB to the Google Play internal track. This requires:
1. A Google Play Console account with the app registered
2. A service account with **Release Manager** role in Google Play Console
3. The service account JSON stored as `GOOGLE_PLAY_SERVICE_ACCOUNT_JSON` in GitHub secrets

Alternatively, Fastlane `supply` can be used for more advanced Play Store management.

## iOS Release Build — Fastlane (Story 4.3)

### Prerequisites

- Xcode 15+ with the `FlowtiqMobile` scheme marked as **Shared**
- Ruby and Bundler
- An Apple Developer account with Team ID
- An App Store Connect API key (P8 format)
- A private GitHub repository to store Fastlane match certificates

### One-time Fastlane match setup

Run once to initialize certificate storage (requires Apple Developer access):

```bash
cd apps/mobile/ios
bundle install
bundle exec fastlane match init    # configure the match git repo URL
bundle exec fastlane match appstore # generate and store certs for com.vastudeep.flowtiq
```

### Creating the App Store Connect API key

1. App Store Connect → Users and Access → Integrations → API Keys
2. Create a new key with **App Manager** role
3. Download the `.p8` file immediately (can only be downloaded once)
4. Encode for GitHub: `base64 -i AuthKey_XXXXXX.p8 | pbcopy`

### Running the beta lane locally

Set environment variables, then run:

```bash
export APPLE_TEAM_ID="YOUR_TEAM_ID"
export APP_STORE_CONNECT_API_KEY_ID="KEY_ID"
export APP_STORE_CONNECT_API_ISSUER_ID="ISSUER_ID"
export APP_STORE_CONNECT_API_KEY_CONTENT="BASE64_P8_CONTENT"
export MATCH_PASSWORD="MATCH_REPO_ENCRYPTION_PASSWORD"
export MATCH_GIT_URL="https://github.com/yourorg/flowtiq-match-certs"

pnpm --filter @flowtiq/mobile ios:release
```

### GitHub Actions secrets (iOS)

| Secret name | Description |
|---|---|
| `APPLE_TEAM_ID` | Apple Developer Team ID |
| `APP_STORE_CONNECT_API_KEY_ID` | ASC API Key ID |
| `APP_STORE_CONNECT_API_ISSUER_ID` | ASC API Issuer ID |
| `APP_STORE_CONNECT_API_KEY_CONTENT` | ASC API Key `.p8` content, base64 encoded |
| `MATCH_PASSWORD` | Fastlane match repository encryption password |
| `MATCH_GIT_URL` | URL of the private match certificates repository |
| `GOOGLE_SERVICE_INFO_PLIST` | GoogleService-Info.plist contents, base64 encoded |

## Deep Linking Setup (Stories 3.5 + 4.4)

Universal Links (iOS) and App Links (Android) are pre-configured for `flowtiq-admin.vercel.app`.

### iOS Universal Links

The `FlowtiqMobile.entitlements` file declares `applinks:flowtiq-admin.vercel.app`.

The verification file is deployed at:
```
apps/admin-portal/public/.well-known/apple-app-site-association
```

**Before deploying:** Replace `PLACEHOLDER_APPLE_TEAM_ID` in the file with the real Apple Team ID.

**Verify deployment:**
```bash
curl https://flowtiq-admin.vercel.app/.well-known/apple-app-site-association
```
Should return HTTP 200 with `Content-Type: application/json`.

**Testing on device (physical device only — not simulator):**
1. Install the app on an iPhone
2. Open Safari and navigate to `https://flowtiq-admin.vercel.app/projects`
3. iOS should offer to open in the Vastudeep Flowtiq app

### Android App Links

The `AndroidManifest.xml` includes an `intent-filter` with `android:autoVerify="true"` for `flowtiq-admin.vercel.app`.

The verification file is deployed at:
```
apps/admin-portal/public/.well-known/assetlinks.json
```

**Before deploying:** Replace `PLACEHOLDER_SHA256_FINGERPRINT` with the actual SHA-256 fingerprint of the release keystore (see extraction command in Android section above).

**Verify deployment:**
```bash
curl https://flowtiq-admin.vercel.app/.well-known/assetlinks.json
```

## CI/CD Pipeline (Story 4.5)

The `.github/workflows/mobile-release.yml` pipeline triggers on:
- Tag push matching `mobile/<tenant>/v<semver>` — automatic
- Manual trigger via GitHub Actions UI

### Triggering a release

```bash
git tag mobile/vastudeep/v1.0.0
git push origin mobile/vastudeep/v1.0.0
```

This runs both `android-build` and `ios-build` jobs in parallel. Both jobs:
1. Apply tenant config
2. Place Firebase credentials from secrets
3. Build and sign the app
4. Submit: Android → Google Play internal track, iOS → TestFlight

### Build duration (approximate)

- Android build: ~10–15 minutes (Gradle + Play Store upload)
- iOS build: ~20–30 minutes (pod install + Fastlane gym + TestFlight upload)
- Total pipeline (parallel): ~30–35 minutes
